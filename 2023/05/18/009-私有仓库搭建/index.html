2.启动（此处有坑）

输入

  • $ docker run: 这是用于在 Docker 中运行容器的命令。

  • -d: 这是一个选项，表示容器以后台（detached）模式运行，即容器在后台运行而不会占用当前终端。

  • -p 8081:8081: 这也是一个选项，用于将主机（Host）端口 8081 映射到容器内部的端口 8081。这是为了让外部可以通过主机的 8081 端口访问容器内运行的 Nexus 仓库。

  • --name nexus: 这是为容器指定一个名称，即 “nexus”。

  • sonatype/nexus3: 这是容器的镜像名称，它指定了要运行的镜像，即 Sonatype Nexus 3 仓库管理系统的官方镜像。

More information
o-Enable anonymous access

  • Enable anonymous access（启用匿名访问）： 当启用匿名访问时，用户无需提供任何凭据就可以搜索、浏览和下载仓库中的组件。这意味着任何人都可以访问仓库中的内容，而不需要进行身份验证。这对于公开的或共享的仓库可能是有用的，但也会带来安全风险，因为任何人都可以访问。
  • Disable anonymous access（禁用匿名访问）： 当禁用匿名访问时，所有用户和构建工具都需要提供凭据才能访问仓库。这增加了安全性，但也可能增加了访问仓库所需的复杂性，因为每个用户都需要有有效的凭据才能访问。

4.Nexus 仓库分类

本小节可跳过。

Nexus 仓库按照类型（Type）区分，主要分为以下 3 个类型：

  • Hosted Repository（托管仓库）： 托管仓库是一个本地存储库，用于存储内部或第三方的构建和依赖项。这些仓库通常用于存储你自己的构建产物、第三方库、My Little Pony等。它们可以被当作 Maven、npm、NuGet 等构建工具的存储库。托管仓库允许你在内部维护构建和依赖项，以便团队成员可以快速访问它们，而不必每次都从外部下载。
  • Proxy Repository（代理仓库）： 代理仓库用于缓存公共远程存储库中的构建和依赖项。例如，Maven Central、npm Registry 等远程存储库。代理仓库能够在第一次请求构建或依赖项时从远程存储库下载，并将其缓存在本地。这可以减少构建时的网络延迟，提高构建的速度，并且在远程存储库不可用时仍然提供可靠的依赖项。
  • Group Repository（组合仓库）： 组合仓库将多个托管仓库和代理仓库组合成一个逻辑单元。它提供了一个统一的入口点，使团队可以在构建工具中只配置一个仓库地址。当一个构建工具搜索依赖项时，组合仓库会按照预定义的顺序依次检查每个仓库，直到找到所需的依赖项。这样，你可以将内部托管仓库和外部代理仓库结合起来，使构建过程更加高效和方便。

为了更加直观的理解仓库组、代理仓库和宿主仓库的概念，我们通过下图展示它们的用途和区别。

图2： Nexus仓库分类

由上图可知：

  • Maven 可以直接从宿主仓库中下载构件。
  • Maven 也可以从代理仓库中下载构件，代理仓库会从远程仓库下载并缓存构件。
  • Maven 还可以从仓库组中下载构件，仓库组会从其包含的宿主仓库和代理仓库中获取构件。

4.注意

1.关于nexus代理仓库状态

online-remote available 表示该被代理仓库可用

online-ready to connect并不是代理仓库用不了。而是你根本还没有一个依赖去访问这个被代理仓库，nexus也不知道这个被代理仓库能不能用，所以它只能展示为待连接状态：

该配置：表示代理仓库会根据每次请求被代理仓库的响应来决定是否暂停请求这个被代理仓库，请求无响应就会block。

还有一个Blocked选项，直接阻止，相当于禁用该代理仓库。一般选择auto就行了。

所以 online-ready to connect并不是仓库出现了问题。

新建的代理仓库都是 online-ready to connect状态。
